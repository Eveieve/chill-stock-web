# Chill Logistics

Chill Logistics is a **warehouse management system for fresh goods**, designed to streamline warehouse operations and manage inventory efficiently.  
Built with real-world scenarios in mind, it handles everything from user management and stock operations to secure session-based authentication.

## Overview

This project was developed as part of an intensive backend engineering program and reflects my ability to design, implement, and maintain a full-featured backend application with a scalable architecture.

Key areas of focus included:

- User authentication and authorization
- Warehouse and stock management
- Admin-level user management and access control
- Responsive and dynamic web views with Thymeleaf

## Tech Stack

- **Spring Framework (MVC architecture)**
- **MyBatis (SQL Mapper Framework)**
- **Thymeleaf (Server-side rendering templates)**
- **MySQL (Relational Database)**
- **Redis (Session storage)**
- **JUnit (Unit testing)**
- **Git + GitHub (Version control & team collaboration)**

## Troubleshooting and Problem Solving

Throughout development, I encountered and solved various technical challenges that enhanced my problem-solving skills:

- **Session Stability:** Integrated Redis to resolve unstable login/logout behaviors in multi-user scenarios.
- **MyBatis Mapping Errors:** Fixed incorrect namespace mappings and parameter mismatches in SQL queries.
- **Entity Relationship Issues:** Handled foreign key constraints and null pointer exceptions during user registration.
- **Thymeleaf Binding Errors:** Resolved model attribute and form submission issues.
- **Transaction Handling:** Addressed partial commits and rollbacks to ensure data consistency.
- **Validation and Error Handling:** Built global exception handlers and frontend + backend validation layers.
- **Version Control Conflicts:** Managed feature branches, resolved merge conflicts, and maintained clean Git history through Pull Requests.

## Getting Started

To run this project locally:

1. Clone the repository
2. Set up MySQL and Redis
3. Configure `application.properties` with your database and Redis credentials
4. Run the Spring application
5. Access the web interface via your browser
